# Sticky Architecture

## Note ownership

- Rust is the sole owner of durable note state. `NoteRepository` serializes every read-modify-write operation behind one mutex and atomically replaces `notes.json` only after the temporary file has been flushed.
- Each note has a stable UUID and stores Tiptap JSON, color, logical position, expanded dimensions, collapsed state, pin state, and its own font size. Window labels derive from the UUID and are not persistence identities themselves. Font-size changes also update the default used by subsequently created notes.
- The webview owns only the live Tiptap editor instance. It sends complete document snapshots to Rust on debounced edits and flushes them at blur, fold, close, and application-quit boundaries.
- Clipboard interoperability uses Tiptap's schema-backed defaults: copy provides HTML and plain-text representations, while paste retains supported semantic structure and discards unsupported markup.

## Snapshots and recovery

- Before each normal replacement, the last valid `notes.json` is atomically retained as `notes.previous.json`.
- If `notes.json` is unreadable, its exact bytes are preserved under `backups/`. A valid previous snapshot is restored when available; an unreadable previous snapshot is also preserved exactly.
- Recovery adds a visible notice note. With no valid snapshot, that notice is the only note in a fresh store.
- Only a brand-new store receives an initial empty note. A valid empty store represents intentional user state and remains empty across restarts.

## Timer ownership

- Rust is the sole owner of durable timer state. `TimerRepository` persists a versioned `timers.json` Tauri store independently from `NoteRepository` and `notes.json`; each timer stores its UUID, accumulated milliseconds, optional absolute running-start timestamp, recurring beep interval, one-time alarm target, pin state, logical position, and folded state after every user mutation. Version 1 timers migrate as expanded.
- A malformed or unsupported timer store is moved intact into `backups/` and replaced with an empty timer store. Timer load or restore failures are logged and skipped without preventing notes from opening.
- One native Rust worker derives elapsed time from the durable accumulated interval and absolute running-start timestamp, so running timers include sleep and time while Sticky was closed without writing to disk each second. It emits display-only ticks to timer windows. Recurring beep schedules always begin at the next future elapsed-time multiple; a worker tick that crosses one or many boundaries plays the macOS Pong alert (`Morse`) once and advances directly to the next future multiple. A configured alarm rotates three preloaded instances of the bundled `codex-notification.wav` through seven groups of three playback starts, 350ms apart within each group and followed by 900ms of silence after the third playback finishes. Simultaneous alarms coalesce, another sequence cannot start while one is active, counting continues afterward, and Reset or moving elapsed time before the target re-arms it. Relaunch never produces a catch-up beep or alarm.

## Window lifecycle

- Expanded dimensions remain canonical while a note is collapsed. Rust performs fold and unfold resizing, and clamps an expanded note to the display containing its collapsed header. Each expanded-note font-size step adjusts window height only by the measured change in content height, preserving its existing visible capacity and making the reverse step restore the prior height. A net increase made while folded grows the note once after unfolding.
- Note dimensions change only through explicit user resizing and lifecycle operations such as fold, unfold, reset, and font-size accommodation. Ordinary typing and paste keep the current window dimensions and scroll inside the note when necessary.
- Rust applies each note's or timer's independently owned durable pin state through one native floating-window helper and macOS collection behavior. Pinned surfaces join every macOS Space and other applications' Stage Manager sets, while unpinned surfaces retain the app's normal transient behavior. Sticky also atomically publishes its open pinned note and timer native window IDs in `aerospace-pinned-windows.tsv`; timer identities use a `timer:` prefix without changing the versioned TSV format. The local AeroSpace workspace hook mirrors those individual windows to the focused AeroSpace workspace without focusing them. As an app-wide override, Ctrl+Cmd+S while a Sticky note is focused toggles every current and future Sticky window together; disabling it leaves individually pinned surfaces active. Each surface's pin button remains the owner of its durable pin state.
- macOS title-bar drags are anchored by Rust to the target native window and current cursor position; frontend code only initiates the drag gesture.
- New notes use the focused note as their anchor and persist the closest edge-aligned position that fits within its monitor work area without overlapping another note.
- Cmd+Shift+N creates a fixed, undecorated timer near the focused note or timer in the closest non-overlapping position on that display. While paused, its large hours, minutes, and seconds display becomes three directly editable fields in Tab order; each completed edit is durably applied as one elapsed-time update and reschedules reminders after the new elapsed value. Timer windows restore unfocused after notes, are clamped into a display work area by native window creation, and persist the resulting position. Their frontend is selected only by `__TIMER_INIT__`; timers share linked-group geometry and drag settlement with notes but do not participate in editor, note archival, note visibility, note repositioning during reset, or coordinated quit-save paths.
- Rust owns a live physical-geometry index for every open note, seeded at window creation and synchronously updated by native move, resize, and destruction events plus programmatic geometry changes. Geometry settlement waits until the primary mouse button is released. Only a completed Sticky title-bar drag may persist a native position or detach a group member; externally imposed moves such as AeroSpace's off-screen workspace hiding never become durable geometry, and pinned notes are returned to their durable screen coordinates. Native resize settlement remains size-only, while editor saves persist only document and color. Group-changing native operations are serialized, rolled back together on failure, and seed both the live geometry and programmatic-move tracking from their authoritative requested targets.
- The note/layout store is the sole owner of linked-group membership. Version 4 groups have durable stable IDs and ordered typed note-or-timer slots, and each window can belong to at most one group; version 3 note-only slots migrate in place without reordering. Relink All Windows on This Side Below Current Window (Cmd+Shift+L or either surface's link button) keeps the initiating window stationary, makes it the parent, and merges all active notes, open timers, and intersecting durable groups on the same half of the same monitor. Other active members are visually ordered beneath it with their sizes preserved and 12px gaps; archived notes and temporarily unavailable timer references from absorbed groups retain their relative slots. Groups on other monitor halves or monitors do not move. Folding, unfolding, note height changes, and font-driven note height changes immediately reflow only later visible members. Dragging a member away detaches and moves only that window, leaving every untouched member in place. Closing a note compacts visible members while retaining its archived slot; restoring or relaunching returns it to that slot. Closing a timer permanently removes its slot and compacts the group. Unlink This Group removes only membership and keeps current geometry. Reset Note Positions clears every mixed group while repositioning only active notes and leaving timers untouched. A group taller than its display remains contiguous but is not forced wholly onscreen.
- Keyboard shortcut help uses a dedicated non-note window toggled by Help → Keyboard Shortcuts or Cmd+/. Esc also closes it. The help window is excluded from note ordering and coordinated note-save shutdown.
- Cmd+Shift+H toggles the visibility of all open note windows without changing their durable note state; restoring the notes returns focus to the previously focused note when it is still open.
- Cmd+Shift+R reopens any missing active note window, resets every active note into a visible cascade inside the primary display's work area, and persists the recovered positions without reopening archived notes or changing note sizes.
- Beginner installation starts with `scripts/bootstrap-macos.sh`, which prepares Apple's compiler tools, clones or updates `~/StickyMD`, and delegates to the root `install.sh`. The root installer checks Apple Silicon, installs checksum-verified official Node and Rust toolchains under the ignored `.tools/` directory, and fingerprints the locked frontend manifests plus its private Node and npm versions so unchanged dependencies survive later updates. It delegates the required release app build and safe replacement to `scripts/install-macos.sh`; developer and release validation plus ZIP creation remain owned by `scripts/package-macos.sh` and are not repeated during an end-user install. Neither installer modifies a user's existing Node, Rust, or shell configuration. Before requesting the running app's final save, each install atomically replaces one bounded `notes.preinstall.json` safety snapshot beside the durable note store; rebuilds never accumulate installer backups.
- Each compiled app embeds the repository's full commit SHA as its build identity. Help → Update opens one dedicated window and is the only automatic trigger for an update check; closing and reopening it starts a fresh check. The window compares that SHA with GitHub's public `main` commit, treats any difference as an available update, and never presents a failed or unknown comparison as up to date.
- Starting an available update opens the fixed `~/StickyMD/scripts/update.command` wrapper in Terminal. The wrapper accepts no arguments and delegates to the existing source installer, which remains the sole owner of source refresh, building, final note saving, app replacement, rollback, and relaunch.
- Every native exit request is blocked by an explicit quit state machine until each note webview acknowledges a successful save; only the coordinator's final exit is permitted.
- Cmd+W dispatches only to the focused window: notes retain their archive-and-close behavior, while timers stop, are deleted permanently from `timers.json`, have any linked-group slot removed, and close. Application-wide emits remain reserved for coordinated note save-on-quit.
- Closing a note archives it with a durable timestamp before closing its window. Archived notes remain recoverable for 30 days, are skipped at launch, and are purged from the current store on the first launch after they expire. Cmd+Shift+T restores the most recent archived note; Cmd+Shift+U atomically restores every archived note and reopens any active note whose window is missing.
- Duplicate launches and macOS Dock reopens reopen any missing active note windows before focusing them, creating a note only when durable state has no active notes.
- The saved autostart preference is reconciled against the launch agent on every release launch. Development builds disable login-item registration.
- Updates remain user-triggered source builds. Local bundles use ad-hoc signing; notarization and public distribution require a separate release decision.
